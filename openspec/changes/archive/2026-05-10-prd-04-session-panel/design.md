## Context

在 `PRD-02` 和 `PRD-03` 完成后，桌宠已经能实时展示主状态并进行关键提醒，但用户仍然缺少查看当前任务摘要、最近活动和其他会话的入口。桥接服务已经提供 `/state`、`/sessions/:id` 和最近事件数据，因此 `PRD-04` 主要是渲染层的结构补齐。

## Goals / Non-Goals

**Goals:**

- 点击桌宠后能打开一个紧凑面板。
- 面板中展示活跃会话摘要、最近活动列表和最近会话列表。
- 支持用户切换查看某个会话详情。
- 保持主桌宠仍由最近活跃会话驱动。

**Non-Goals:**

- 不实现聊天输入框。
- 不实现复杂过滤、搜索和分页。
- 不实现将主桌宠固定到非活跃会话。

## Decisions

### 1. 面板状态与主桌宠状态分离

主桌宠继续直接消费 `useBridgeState` 的活跃会话，而面板额外维护 `isOpen`、`selectedSessionId` 和可选的 `selectedSessionDetail`。这样可以保证查看其他会话时，不会影响主桌宠当前状态。

### 2. 详情数据按需请求

活跃会话详情可以直接从已有桥接快照和最近事件推导，非活跃会话则通过 `/sessions/:id` 按需拉取详情。这样既能保持数据准确，也避免在第一版提前把所有会话完整详情常驻在前端内存里。

### 3. 敏感信息只展示摘要

面板只展示事件 `summary`、`title`、状态和时间，不直接展示原始日志或大段详情，符合第一版对敏感日志的约束。

## Risks / Trade-offs

- [面板信息过多导致主界面拥挤] -> 保持紧凑布局，并按“活跃会话 / 最近活动 / 最近会话”分块。
- [非活跃会话详情与实时状态存在短暂不一致] -> 切换会话时拉取详情，并在桥接快照更新时刷新活跃会话内容。
- [提醒层与面板层遮挡] -> 面板使用单独容器，保持提醒层优先级更高。

## Migration Plan

- 先补一个面板状态 hook 和会话详情按需拉取逻辑。
- 再接入 `PetShell` 布局，增加点击开关和面板组件。
- 最后通过本地模拟验证多会话列表、切换和最近活动显示。
