# bridge

`apps/bridge` 是桌宠命令行助手的本地桥接服务。

## 作用

- 接收适配器或其他本地组件上报的标准事件。
- 保存最近事件历史，供桌面端查询。
- 通过 WebSocket 将最新事件广播给桌面端。
- 作为后续会话状态管理、动作审计和动作路由的承载点。

## 当前实现

当前入口文件为 `src/index.ts`，已经具备：

- `GET /health`：健康检查。
- `GET /events`：返回最近事件列表。
- `POST /events`：接收事件并做协议归一化。
- `WS /events`：向客户端广播最新事件。

服务默认只绑定 `127.0.0.1`，符合项目的本地安全边界。

## 依赖关系

- 输入依赖：`@termpet/protocol`
- 输出对象：`apps/desktop`
