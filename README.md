# 桌宠命令行助手

桌宠命令行助手是一个面向命令行工具和代码代理工作流的桌面桌宠。它通过二维动态角色表达任务状态，让用户不用一直盯着终端，也能快速知道任务是在思考、工作、等待确认、完成，还是失败。

第一阶段先聚焦代码代理工具的状态可视化。整体架构会从一开始预留插件能力，后续可以适配更多命令行工具。

## 产品定位

本项目同时是效率工具和陪伴式界面。

- 效率：减少用户盯着长时间运行任务的成本。
- 感知：把重要状态变化、失败结果、确认请求及时展示出来。
- 陪伴：用可配置的二维动态角色和角色灵魂表达任务状态。
- 扩展：先支持代码代理工具，后续通过适配器支持更多命令行工具。

## 第一版范围

第一版要先跑通核心链路：

```text
代码代理事件
  -> 本地桥接服务
  -> 标准桌宠事件
  -> 桌面角色状态变化
  -> 必要时打断提醒或展示操作面板
```

第一版能力：

- 透明、置顶、可拖拽的桌面桌宠窗口。
- 支持 `空闲`、`监听`、`思考`、`工作`、`等待确认`、`成功`、`失败` 等状态。
- 通过代码代理工具的钩子能力接入状态事件。
- 本地桥接服务负责接收事件，并广播给桌面端。
- 点击桌宠后展示当前任务、最近日志、快捷操作和设置入口。
- 支持多会话状态跟踪，主桌宠默认展示最近活跃会话。
- 对权限确认请求进行明显提醒。

## 第一版暂不实现

以下能力先预留设计，不放进第一版：

- 自动识别所有终端内容。
- 适配其他代码代理、包管理器、版本控制、测试框架、容器工具等。
- 插件市场。
- 大模型生成角色灵魂行为。
- 远程会话同步。
- 完整聊天界面。

## 设计原则

- 桌宠首先要表达状态，而不是制造噪音。
- 需要确认、失败、完成这类重要事件必须清晰打断。
- 桌面端不直接理解具体命令行工具的内部结构。
- 适配器负责把工具自己的事件转换成统一桌宠协议。
- 协议从第一天开始支持多会话。
- 敏感终端细节默认隐藏，只有用户主动查看时才展示。

## 第一版技术建议

第一版默认技术选型：

- 包管理器：`pnpm`。
- 工作区：`pnpm workspace`。
- 桌面端：`Electron`。
- 主要语言：TypeScript。
- 样式方案：Tailwind CSS。
- 桌面端和桥接服务通信：本地网页套接字。
- 适配器到桥接服务通信：本地超文本接口。

第一版选择 `Electron`，因为透明窗口、无边框窗口、置顶窗口和网页式二维动态模型渲染都比较直接。后续如果安装包体积和资源占用成为主要问题，再评估更轻量的桌面方案。

第一版可运行原型应包含：

- `apps/bridge`：本地事件接收和广播服务。
- `apps/desktop`：透明桌宠窗口和任务面板。
- `plugins/codex`：代码代理事件映射。
- `packages/protocol`：共享事件类型。
