# 路线图

## 阶段 0：文档和项目骨架

目标：

- 明确产品范围。
- 明确第一版产品需求。
- 明确事件协议。
- 明确插件系统。
- 明确角色灵魂系统。
- 明确安全与隐私规则。

交付物：

- 根目录说明文档。
- 产品需求文档。
- 架构文档。
- 协议文档。
- 插件文档。
- 路线图。
- 明确 `pnpm`、`pnpm workspace`、`Electron`、TypeScript、Tailwind CSS 作为第一版默认技术选型。

## 阶段 1：代码代理可视化第一版

目标：

- 连接代码代理事件和桌面桌宠。
- 验证二维动态角色状态可视化。
- 支持需要注意的主动提醒。

交付物：

- 基于 `Electron` 的桌面端应用。
- 透明置顶桌宠窗口。
- 基础二维动态模型加载。
- 状态到动作映射。
- 本地桥接服务。
- 代码代理钩子事件接收器。
- 最近活跃会话跟踪。
- 点击面板，包含任务详情、最近日志、快捷操作和设置入口。
- 等待确认提醒弹窗。

成功标准：

- 代码代理状态变化能驱动桌宠表情。
- 用户能感知思考、工作、等待确认、成功、失败状态。
- 点击桌宠能看到有用的会话上下文。

## 阶段 2：稳定性和设置

目标：

- 让第一版可以日常使用。
- 降低通知噪音，同时保留重要提醒。

交付物：

- 持久化应用设置。
- 桌宠窗口位置持久化。
- 通知控制。
- 最近事件历史。
- 带保留上限的原始载荷存储。
- 桥接服务或二维动态模型失败时的降级处理。

## 阶段 3：插件边界

目标：

- 把代码代理适配器抽成正式插件。
- 为非代码代理命令行工具支持做准备。

交付物：

- 插件清单加载器。
- 插件开发接口。
- 代码代理插件作为第一方插件打包。
- 权限声明模型。
- 插件启用和禁用设置。

## 阶段 4：通用命令包装器

目标：

- 通过显式包装器支持任意命令行命令。

交付物：

- `termpet run <command>` 命令。
- 进程生命周期跟踪。
- 带隐私控制的标准输出和错误输出捕获。
- 基于退出码的成功和失败事件。
- 针对包管理器、版本控制、测试命令的基础规则。

## 阶段 5：角色和灵魂扩展

目标：

- 让角色行为可配置、更有表达力。

交付物：

- 角色包清单。
- 角色灵魂配置编辑器。
- 动作映射编辑器。
- 多角色包。
- 角色包导入和导出。

## 阶段 6：大模型角色灵魂

目标：

- 让角色根据脱敏事件摘要生成短状态文案和性格化表达。

交付物：

- 大模型角色灵魂提示词模板。
- 隐私控制。
- 短文案生成。
- 语气和性格配置。
- 后续支持本地或云端模型选择。

## 阶段 7：更多命令行工具适配器

可能适配：

- 其他代码代理工具。
- 包管理器。
- 版本控制工具。
- 测试框架。
- 容器工具。
- 远程会话或远程桥接同步。

## 阶段 8：分发

目标：

- 面向真实用户发布桌宠命令行助手。

交付物：

- Windows 安装包。
- 自动更新策略。
- 首次启动设置流程。
- 代码代理钩子设置助手。
- 插件权限界面。
- 角色包导入界面。
