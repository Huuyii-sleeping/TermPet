# 安全与隐私

## 原则

桌宠命令行助手会观察命令行工具和代码代理活动，因此可能接触路径、源码、命令输出、环境信息和确认请求。默认行为必须保守。

核心原则：

- 默认只展示短摘要。
- 原始输出默认隐藏，只有用户主动查看时才展示。
- 最近日志采用有限保留。
- 确认动作属于安全敏感操作。
- 插件权限必须可见。
- 除非用户明确开启，否则不要把敏感数据发送给大模型服务。

## 敏感数据

潜在敏感内容：

- 接口密钥和令牌。
- 环境变量。
- 私有源码。
- 文件路径。
- 命令输出。
- 版本控制远程地址。
- 错误日志。
- 用户提示词。

## 事件脱敏

适配器应输出简短摘要。

推荐：

```json
{
  "title": "代码代理需要确认",
  "summary": "代码代理请求运行一个命令。",
  "rawRef": "raw_evt_abc"
}
```

避免：

```json
{
  "summary": "代码代理请求运行：curl -H Authorization: Bearer sk-..."
}
```

## 原始载荷存储

原始载荷应该：

- 只存储在本地。
- 只保留较短的有限时间或有限数量。
- 通过标识引用。
- 不出现在气泡和轻提示里。
- 只在详情面板中由用户主动查看。

## 确认安全

当桌宠展示确认弹窗时，必须展示：

- 来源工具。
- 工作目录。
- 请求动作类别。
- 风险级别。
- 短摘要。
- 查看详情按钮。
- 在支持时显示允许和拒绝按钮。

中高风险动作必须要求用户明确点击确认。除非用户主动开启，否则不应该支持键盘快捷确认。

风险示例：

| 风险 | 示例 |
| --- | --- |
| 低 | 读取文件、列出目录、运行不修改状态的检查命令。 |
| 中 | 修改文件、运行测试、安装本地依赖。 |
| 高 | 删除文件、网络请求、凭据相关命令、推送代码。 |

## 插件权限

插件应声明权限：

- `receive_hook_payload`：接收钩子载荷。
- `read_workspace_path`：读取工作目录。
- `read_command_output`：读取命令输出。
- `store_raw_logs`：存储原始日志。
- `send_local_events`：发送本地事件。
- `send_actions_to_cli`：向命令行工具回传动作。
- `network_access`：访问网络。

第一版可以内置代码代理插件，但权限模型应先记录在文档中，并在后续设置界面体现。

## 大模型角色灵魂隐私

大模型生成角色灵魂行为放在第一版之后。加入时应遵守：

- 默认不发送原始日志。
- 默认只使用脱敏摘要。
- 发送命令输出必须由用户明确开启。
- 后续可支持本地模型模式。
- 未经明确确认，不发送权限请求详情。

## 本地网络

桥接服务默认只绑定本机地址。

推荐：

```text
127.0.0.1
```

除非用户明确开启远程同步，否则不要把桥接接口暴露到局域网。

## 审计日志

桥接服务应记录：

- 确认请求事件。
- 用户决定。
- 动作路由失败。
- 插件启用或禁用。
- 安全设置变更。

审计日志应避免存储完整原始命令输出，除非用户开启调试模式。
