# PRD-04 点击面板与多会话展示

## 目标

把桌宠从“只看状态”提升到“可查看上下文”，补齐点击面板与多会话展示能力。

## 解决的问题

- 当前用户无法从桌宠上查看当前任务细节。
- 多会话虽然在目标里存在，但还没有可见入口。
- 最近事件和会话摘要没有统一展示界面。

## 范围

- 点击桌宠打开紧凑面板。
- 展示当前活跃会话摘要。
- 展示最近事件或最近日志摘要。
- 展示最近会话列表。
- 允许用户切换查看某个会话详情。

## 不在本 PRD 范围内

- 聊天输入框。
- 复杂过滤、搜索和分页。

## 主要交付

- 面板开关与布局。
- 活跃会话卡片。
- 最近活动列表。
- 多会话列表与切换行为。
- 设置入口占位。

## 实现要求

- 主桌宠仍默认跟随最近活跃会话。
- 面板里可以查看其他会话，但第一版不要求“固定主桌宠到某会话”。
- 敏感原始日志默认不直接展示，只展示摘要。

## 验收标准

- 用户点击桌宠后能看到当前任务标题、摘要和最近活动。
- 当存在多个会话时，面板能列出并切换查看。
- 主桌宠状态仍然遵循最近活跃会话规则。

## 对后续 PRD 的依赖价值

- `prd-06` 会把面板设置和窗口偏好持久化。
