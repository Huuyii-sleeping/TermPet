# PRD-05 Codex 适配器增强与动作链路

## 目标

把当前较粗糙的 Codex 事件映射，补成第一版可用的适配器，并预留动作回传链路。

## 解决的问题

- 现有事件映射覆盖面有限，语义也较粗。
- `PostToolUse` 成功与失败未充分区分。
- 权限确认虽然能转成等待状态，但还没有动作链路模型。

## 范围

- 细化 Codex 生命周期事件到标准状态的映射。
- 为确认、拒绝、查看详情、回终端操作生成标准动作。
- 桥接服务内接入最小动作路由接口。
- 在不支持真实回传时，保留终端回退模式。

## 不在本 PRD 范围内

- 适配其他命令行工具。
- 完整插件市场。

## 主要交付

- 更完整的 Codex hook 映射表。
- 标准动作生成逻辑。
- 动作请求与动作结果事件模型落地。
- 最小可用的桥接动作接口。

## 实现要求

- 区分成功停止、失败停止和人工中断。
- 权限请求必须尽可能带上来源、目录、摘要和风险等级。
- 敏感原始载荷只保存本地引用，不进入普通摘要。

## 验收标准

- 常见 Codex 生命周期事件能稳定转换为标准协议事件。
- 权限请求事件能带出标准动作列表。
- 无论是否支持真实回传，用户都能获得清晰的下一步操作方式。

## 对后续 PRD 的依赖价值

- `prd-06` 将基于动作链路补审计和持久化。
