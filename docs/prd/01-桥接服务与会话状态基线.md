# PRD-01 桥接服务与会话状态基线

## 目标

把当前“仅广播事件”的桥接原型补到“可作为桌宠状态单一事实来源”的基础版本。

## 解决的问题

- 现在桥接服务只有最近事件列表，没有稳定的会话状态聚合。
- 桌面端无法可靠获取“当前主会话”和“最近会话列表”。
- 后续打断提醒、点击面板和动作路由都缺少统一状态基线。

## 范围

- 基于 `sessionId` 维护会话状态。
- 维护最近活跃会话排序。
- 提供当前活跃会话、会话列表和最近事件查询接口。
- 为动作审计和动作路由预留最小数据结构。

## 不在本 PRD 范围内

- 桌宠 UI 细节。
- 真正的允许/拒绝动作回传。
- 复杂持久化。

## 主要交付

- 桥接服务内部 `session store`。
- 最近事件与最近会话的统一内存模型。
- `GET /sessions`
- `GET /sessions/:id`
- `GET /events`
- WebSocket 广播包含标准事件与活跃会话变化。

## 实现要求

- 所有状态更新以标准协议事件为输入。
- 主活跃会话默认取最近活动会话。
- 对重复事件具备幂等或最小重复容忍能力。
- 历史保留上限可配置，默认保留有限数量。

## 验收标准

- 连续发送多个 `sessionId` 的事件后，桥接服务能返回正确的会话列表。
- 最新活跃会话会随着新事件自动切换。
- 同一会话的 `currentState`、`latestEventId`、`latestActivityAt` 能正确更新。
- 桌面端可以通过桥接接口获得稳定、可复现的会话视图。

## 对后续 PRD 的依赖价值

- `prd-02` 依赖它驱动真实桌宠状态。
- `prd-04` 依赖它展示多会话面板。
- `prd-05` 依赖它挂接动作路由和审计。
