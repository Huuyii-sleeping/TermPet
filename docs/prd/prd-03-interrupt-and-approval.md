# PRD-03 关键状态打断与确认提醒

## 目标

让桌宠具备第一版最关键的主动提醒能力，重点覆盖等待确认、失败和完成状态。

## 解决的问题

- 当前没有真正的打断层级体验。
- 用户可能错过权限确认和失败事件。
- 成功、失败、等待确认三类状态还没有清晰区别。

## 范围

- 按协议中的 `interruptLevel` 实现基础提醒表现。
- `waiting_approval` 使用强提醒。
- `error` 使用强提醒并保留详情入口。
- `success` 使用轻提醒。
- 无法直接回传时，明确提示用户回终端确认。

## 不在本 PRD 范围内

- 真正的动作执行回传。
- 完整设置系统。

## 主要交付

- `silent` / `bubble` / `toast` / `modal` 的最小 UI 实现。
- 等待确认提醒卡片或弹窗。
- 失败提醒与最近摘要入口。
- 成功完成提示。

## 实现要求

- 风险信息不能藏在装饰性视觉里。
- 强提醒必须显示来源工具、工作目录、动作摘要和详情入口。
- 在无法直接执行确认动作时，必须显示终端回退提示。

## 验收标准

- 权限请求到达后，用户无需盯终端也能明显看到提醒。
- 错误事件到达后，用户能快速知道是失败而不是普通状态切换。
- 成功事件到达后，会显示轻量完成提示，不长期阻塞桌面。

## 对后续 PRD 的依赖价值

- `prd-05` 会在此基础上接入真实动作回传。
- `prd-06` 会将提醒强度纳入设置持久化。
