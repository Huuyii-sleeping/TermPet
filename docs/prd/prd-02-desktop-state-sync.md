# PRD-02 桌宠状态渲染与实时同步

## 目标

把当前基于 `demoState` 的桌宠原型，改成由桥接服务实时驱动的可运行桌宠。

## 解决的问题

- 当前桌宠状态是硬编码的，无法反映真实事件。
- 渲染层虽然已连接 WebSocket，但没有写入 React 状态。
- 角色动作映射没有真正接入运行时状态流。

## 范围

- 桌面端启动时拉取初始状态。
- 通过 WebSocket 接收增量事件并更新当前主会话状态。
- 使用真实状态驱动桌宠文案、动作和表情。
- 在桥接断开和恢复时具备基本重连行为。

## 不在本 PRD 范围内

- 点击面板完整交互。
- 确认弹窗和高优先级打断。
- 多会话切换 UI。

## 主要交付

- 渲染层状态容器。
- “初始查询 + 增量订阅” 同步流程。
- 当前主状态卡片。
- 状态到动作/表情映射的真实接入。

## 实现要求

- 启动后先拉取当前会话与最近事件，再建立 WebSocket 订阅。
- 当活跃会话变化时，桌宠主视图立即切换。
- 断线时给出轻量降级状态，不影响窗口继续存在。

## 验收标准

- 当桥接服务收到 `thinking`、`working`、`waiting_approval`、`success`、`error` 事件时，桌宠能立即切换显示。
- 桥接服务重启后，桌面端可以重新连接并恢复状态。
- 移除 `demoState` 后仍能完整运行桌宠窗口。

## 对后续 PRD 的依赖价值

- `prd-03` 基于此增加打断提醒。
- `prd-04` 基于此补完整面板与会话展示。
